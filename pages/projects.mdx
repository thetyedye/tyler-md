# Projects

Things I'm building. Day-job project first, then personal stuff.

## The Lab
*Status: building · Strategy Labs · day job*

An agentic operating system for Strategy Labs. A shared brain, a data layer, and workflow skills that make the agency AI-native — agents that know our clients, playbooks, and history, and pull source-backed answers instead of hallucinating them.

Three pieces:

- **Agency brain** — client vaults, playbooks, people and company profiles, decision history, call transcripts. Consolidated from Slack, ClickUp, Fireflies, Gmail, Drive, and Funnel.io.
- **Agent-queryable data layer** — Funnel.io → Supabase, exposed through MCP tools so agents can pull live performance numbers without humans rebuilding reports.
- **Workflow skills** — repeatable client work mapped into agent-runnable processes with human review checkpoints. First one live: weekly client performance summary, source-backed, reviewed, sent.

The substrate is durable. The interfaces (Claude, Codex, whatever's next) are interchangeable.

## tyler-brain
*Status: ongoing · personal*

My personal knowledge graph in plain markdown. Three layers: `raw/` for frictionless daily capture, `sources/` for curated external material, `wiki/` for compiled knowledge with cross-linked entities (people, companies, projects, ideas, tools). No vector DB, no embeddings — retrieval is `grep` + `read` + `glob`. Every fact has a source. Brain-first: my agents check here before reaching for an external API.

## hermes
*Status: running · personal*

The always-on Telegram agent that brings tyler-agent to life. Dockerized runtime that mounts `tyler-agent` and `tyler-brain` as volumes — code change reflects live, no rebuild. Heartbeat, scheduled skills (morning brief, midday check-in, ambient capture), voice-first via Whispr Flow. Phone-first dictation, source-backed responses, calendar awareness.

## Printing Press CLIs
*Status: ongoing · personal*

A family of token-efficient Go binaries that sync data into SQLite and serve queries offline. Movie discovery, recipe search, ESPN scores, flight search. Used as skills inside the agent stack — instead of burning tokens on web search, the agent calls a local CLI that already knows the answer.
