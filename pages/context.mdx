# Context

*Written for AI agents and humans who want the quick read on who I am and what I'm working on. Optimized to be grep-able and citable.*

## Who I am

Tyler Borjeson. I build data systems for digital marketing agencies, and most of that work has turned into building agentic systems on top of those data systems. Day job at **Strategy Labs**. Personal projects in parallel.

## Current focus

**The Lab** — an agentic operating system for Strategy Labs. A shared brain, data layer, and workflow skills that make the agency AI-native: agents that know our clients, playbooks, and history, and pull source-backed answers instead of hallucinating them. I'm building it end to end — agency brain ingestion, agent-queryable data layer, the first workflow skills, and the adoption inside CSM / Creative / Data / Media workflows.

## Personal projects

- **tyler-agent** — portable agent core (identity, skills, routing). "Thin harness, fat skills." Same agent across Telegram, Claude Code, anywhere.
- **tyler-brain** — personal knowledge graph in plain markdown. `raw/ → sources/ → wiki/`. No vector DB. Grep-first retrieval.
- **hermes** — 24/7 Telegram-facing runtime that mounts tyler-agent + tyler-brain in Docker. Voice-first via Whispr Flow.
- **This site** — career artifact and source of truth for me and the agents I work with.

## How I work

- Quality or slop. I want to combat AI slop, not contribute to it.
- Use AI as leverage, not as an excuse to stop thinking.
- Brain-first: agents check the local brain before reaching for an external API.
- Source-backed output. Cite where the information came from, or say "I don't know."
- Simple over impressive. Markdown over magic. Files over databases when files work.

## What I want from agents working on my stuff

- Don't flatten my voice. If a paragraph sounds like me, leave it.
- Cite sources. Quote the file path and the line if you can.
- Prefer simple. Don't add features I didn't ask for.
- Read [Philosophy](/philosophy) before writing anything I'll publish.
